## Part G — hair-color prompt + 模型选型（4 次迭代）

用户实测 hair-color 发现：**束发被释放成披散长发**。

### Iteration 1 — TASK/DO/DO NOT 三段式 prompt（commit `09304f3`）

老 prompt 把约束埋在 prose 长 list 里，Gemini 漏读。改成：

```
TASK
Recolor only the hair to ${hex}. Nothing else changes.

✅ DO
- [行动 bullets]

❌ DO NOT (any violation = failure; treat as hard rules)
- DO NOT [显式禁止 bullets]

EDGE CASE — OCCLUDED HAIR
[针对束发等特定失败模式的显式约束]
```

实测：成功阻止「释放成披散」的 catastrophic 失败，但 Flash 还是会加面部周围细丝。

**Memory 沉淀**：`feedback_prompt_do_dont_split.md` 写入未来所有 Gemini edit prompt 通用原则。

### Iteration 2 — 用户反驳「不应该枚举发型」

「发型无限，列举不完」是对的——改成**原则性表述**：

```
STRICT INVARIANT
- The hairstyle must NOT change in any way
- Every non-hair pixel must be IDENTICAL to the source  
- The hair-silhouette outline must be IDENTICAL to the source
- ONLY the COLOR of hair pixels may change
- Treat as color-only inpainting operation
```
