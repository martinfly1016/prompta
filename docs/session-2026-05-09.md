## 重大决策与教训

### 1. Quota race condition 修复（核心 bug）

**问题**：5 并发 `/api/tools/hair-color/simulate` 绕过 FREE_LIMIT=3 → DB 创建 4 行（实测）。

**第一次尝试失败**（commit `8d49a94`）：用 `prisma.$transaction(tx => tx.$executeRaw\`SELECT pg_advisory_xact_lock(${key})\`)`。看似合理，但实测在 Prisma + Vercel + Railway PG 环境**完全不阻塞**（5 并发仍然产生 4 行）。原因不明（怀疑 `$executeRaw` 对 SELECT 实际不执行、或 PgBouncer transaction-mode 破坏 advisory lock 跨语句保持）。

**最终修复**（commit `5c8e847`）：rank-after-insert 模式
- INSERT 立即 auto-commit（无 transaction）
- 用 `count` 计算自己 rank（按 createdAt + id tie-break）
- 超额则 DELETE 自己

prod 实测：5 并发 → 严格 3×200 + 2×429 ✓

写入 `feedback_prisma_advisory_lock_avoid.md` 永久记忆。

### 2. GSC 数据假象

早晨日报看到 GSC clicks/impressions −18%，下午重拉同窗口 → 数据从 675 → 808，实际是 −1.7%。**GSC 末 1-2 天数据延迟是常态**，不要立刻定性问题。GA4 是实时的，所以 GA 涨 + GSC 跌的反差大概率是 GSC 滞后。写入 `reference_gsc_data_lag.md`。

### 3. macOS bash 3.2 `& wait` 死锁

测试套件 race case 用 `for i; do (...) &; done; wait` 在 macOS /bin/bash 3.2 下导致 parent stuck 在 `__wait4` syscall（用 `sample` 验证），即使 children 都退出。换成 `xargs -P 5 -I IDX sh -c '...'` 后立即解决。

### 4. ChatGPT 引用 prompta.jp 的 GEO 信号

GA4 数据：chatgpt.com referral W18 3 sessions → W19 17 sessions（**+5.7x**），平均停留 189s / bounce 7.7% — 站内最优质来源。证据是 LLM 时代的新流量渠道。已：
