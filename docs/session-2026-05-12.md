## 关键 feedback / 决策记录

- **Cannibalization 是头部词卡 50-100 位的根因**，不是单页 SEO 弱。同一关键词 4-5 个着陆页争抢的格局必须先分流再讨论排名提升
- **Type-A/B/C cluster 分流策略**：A 定义走 guide 长文 / B 書き方走专门 guide / C 一覧走首页（Prompta 本质是目录站）
- **`TOOL_PAGE_CONTENT` Record 模式**：第二个工具（gemini）以下迁移成本是 0 代码改动，只填数据
- **`CategoryIntro.faqs?` 选项化**：现有 12+ categories 不动，opt-in 即可。两个新 category（writing/cosplay）作为 prototype 验证模式
- **GSC `_head_keyword_pages.ts` 多页查询**才能看出 cannibalization；单纯 `top-queries` 看不到 query-to-page 1:N 分布
- **`_rank_opportunities.ts` default 28d 而非 7d**：低流量站 7 日窗口几乎无 ≥ 40 曝光的词，GSC UI Performance 报告也默认 28d
- **Request indexing 没公开 API**，但 URL Inspection API 可以验证爬虫已重新访问（lastCrawlTime 字段）
- **GSC 数据 24-48h 延迟原则**仍适用（脚本所有窗口都 end yesterday）
