### 架构（drop-a-file workflow）

**核心流程**：用户改控件 → state 改变 → prompt 内容里对应 token 实时替换 + 高亮 → 「コピー」复制改后版本。SSR 仍渲染原始内容（SEO 不受影响）。

**关键文件**：

```
src/lib/prompt-params/
├── types.ts                      # ParamConfig 类型定义
├── interpolate.ts                # 纯函数：buildSegments / defaultValues
├── options.ts                    # 共享 options：hairColor/Length/Style, expression, eyeColor, fabricColor(suffix)
├── registry.ts                   # 读 ALL_CONFIGS + slug aliases
├── configs.generated.ts          # 自动生成 barrel（不要手改）
└── configs/<slug>.ts             # 每个参数化 prompt 的 config（drop-in）

src/components/prompt/params/
├── PromptParamsPanel.tsx         # client component：state + 实时渲染 + diff highlight
├── ParamColorSwatch.tsx          # 圆形色板，自动对比度 checkmark
└── ParamSelect.tsx               # chip/pill select

scripts/prompt-params/
├── sync.ts                       # 扫 configs/ 重写 configs.generated.ts
└── audit.ts                      # 校验每个 match 在 DB 中存在且唯一

package.json scripts:
  params:sync / params:audit
```

**集成点**：
- `src/app/(marketing)/prompt/[slug]/page.tsx` — `getPromptParamsConfig(slug)` 决定走交互面板还是原静态布局
- `src/lib/track.ts` — `trackParamChange(promptId, paramId, value, meta)` 发 GA `prompt_param_changed`
- `.claude/skills/collect-content/SKILL.md` — Phase 4.5 § ParamConfig 抽取（Haiku 子代理为 tag-style 自动产出 config）

**禁止事项**：
- ❌ 不要在 config 里 copy-paste 12 色 swatch — 用 `hairColorOptions(default)` / `fabricColorOptions(suffix, default)` 等辅助函数
- ❌ 不要手改 `configs.generated.ts` — 跑 `npx tsx scripts/prompt-params/sync.ts`
- ❌ 不要给 narrative 类 prompt（photo-edit / chatgpt 长文）配 ParamConfig — 短 token 替换会破坏上下文
